Data Flow Pattern

This project follows a configuration-driven architecture. Content changes are made in one place.

1. Define in Config

Add/edit features, comparisons, and nav items in improvePageConfig.ts

2. Components Render

Components receive data as props and render dynamically using .map()

3. Assets Referenced

Image paths in config point to static files in /public/assets/

4. Single Source of Truth

Change a title once in config → updates everywhere automatically
